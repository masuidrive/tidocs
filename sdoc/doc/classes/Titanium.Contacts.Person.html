Object Titanium.Contacts.Person

  • Android(0.8+), iPhone(0.8+), iPad(0.8+)

An object that represents a contact record for a person or organization in the system contacts address book.

A person object is created using Titanium.Contacts.createPerson.

The following two kinds of properties exist for this object:

  • single value - contains either a string or number type value, an array of string type values, or null if unset.
  • multi-value - contains a dictionary with typical keys of home, work and/or other. Each key contains either a string type value, an array of string type values, or a dictionary containing key/value pairs with string type values.

Adding and Modifying Properties

Support for adding and modifying properties is currently only supported on iOS.

Keys as Address Book UI Labels

Keys act as labels in the address book user interface.

Although there is limited support for custom, arbitrarily-named, keys when used with multi-value properties, there is no support them with single value properties.

On iOS, creating custom keys is not recommended, and will lead to undefined results.

If a label has been created by the user of the device and used with a multi-value property, it will exist as a key with the same name.

See examples in Titanium.Contacts for more information.

Properties
address [Dictionary] Addresses for the person. Multi-value. Read-only on Android.
birthday [String] Date of birth of the person. Single value.
created [String] Date and time that the person record was created. Single value.
date [Dictionary] Dates associated with the person. Multi-value.
department [String] Department of the person. Single value.
email [Dictionary] Email addresses for the person. Multi-value. Read-only on Android.
firstName [String] First name of the person. Single value.
firstPhonetic [String] Phonetic first name of the person. Single value.
fullName [String] Localized full name of the person. Single value. Read-only on Android.
id [Number] Record identifier of the person. Single value.
image [Titanium.Blob] Image for the person. Single value.
instantMessage [Dictionary] Instant messenger names of the person. Multi-value.
jobTitle [String] Job title of the person. Single value.
kind [Number] Determines the type of information the person record contains; either person or organization. Read-only on Android.
lastName [String] Last name of the person. Single value.
lastPhonetic [String] Phonetic last name of the person. Single value.
middleName [String] Middle name of the person. Single value.
middlePhonetic [String] Phonetic middle name of the person. Single value.
modified [String] Date and time that the person record was last modified. Single value.
nickname [String] Nickname of the person. Single value.
note [String] Notes for the person. Single value.
organization [String] Organization to which the person belongs. Single value.
phone [Dictionary] Phone numbers for the person. Multi-value. Read-only on Android.
prefix [String] Prefix for the person. Single value.
recordId [Number] Record identifier of the person. Single value.
relatedNames [Dictionary] Names of people to which the person is related. Multi-value.
suffix [String] Suffix for the person. Single value. Read-only on Android.
url [Dictionary] URLs of webpages associated with the person. Multi-value.
