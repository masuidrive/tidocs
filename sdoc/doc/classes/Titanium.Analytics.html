Module Titanium.Analytics

  • Android(0.1+), iPhone(0.1+), iPad(0.1+), Mobile Web(1.8+)

Used for transmitting developer-defined Analytics events to the Appcelerator Analytics product.

The analytics module can be used to supply additional context or application-specific information which can then be accessed during analysis using Analytics.

Use the featureEvent method to generate custom events that you can view through the Analytics product. You can specify a name for the feature event which is visible through Analytics.

NOTE Although the analytics module allows you to transmit other types of events, such as navigation events and timed events, data regarding these events is not accessible through the Analytics UI.

Navigation events will be supported in a future version of the Analytics product. There is no plan to support the other event types, and they and should not be used.

Methods
  • addEvent (type, name, data)
  • featureEvent (name, data)
  • navEvent (from, to, name, data)
  • settingsEvent (name, data)
  • timedEvent (name, start, stop, duration, data)
  • userEvent (name, data)
Code Examples
Custom Feature Event

This example shows how to send a feature event during an application session to indicate some feature that you would like to track was used.

Titanium.Analytics.featureEvent('app.feature.blah');

In this case, the Analytics product would show statistics about how many times the 'app.feature.blah' event was generated.

Methods
addEvent (type, name, data) : void

Sends a generic event for this application session. Not displayed in Analytics UI.

Arguments

type [String] Event type.
name [String] Event name.
data [Object] Extra data related to the event. Must be serializable as JSON.
featureEvent (name, data) : void

Sends a feature event for this application session.

Arguments

name [String] Event name, displayed in Analytics UI.
data [Object] Extra data related to the event, not displayed in Analytics UI. The object must be serializable as JSON.
navEvent (from, to, name, data) : void

Sends a navigation event for this application session. Not displayed in Analytics UI.

Arguments

from [String] String describing the location the user navigated from.
to [String] String describing the location the user navigated to.
name [String] Event name.
data [Object] Extra data related to the event. The object must be serializable as JSON.
settingsEvent (name, data) : void

Sends a settings event for this application session. Not displayed in Analytics UI.

Arguments

name [String] Event name.
data [Object] Extra data related to the event. Must be serializable as JSON.
timedEvent (name, start, stop, duration, data) : void

Send a timed event for this application session. Not displayed in Analytics UI.

Arguments

name [String] Event name.
start [Date] Event start as a Date object.
stop [Date] Event end as a Date object.
duration [Number] Event duration.
data [Object] Extra data related to the event. Must be serializable as JSON.
userEvent (name, data) : void

Sends a user event for this application session. Not displayed in Analytics UI.

Arguments

name [String] Event name.
data [Object] Extra data related to the event. Must be serializable as JSON.
