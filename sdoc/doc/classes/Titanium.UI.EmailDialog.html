Object Titanium.UI.EmailDialog

  • Android(0.8+), iPhone(0.8+), iPad(0.8+), Mobile Web(1.8+)

An email dialog is a modal window that allows users to compose and send an email.

The Email Dialog is created by Titanium.UI.createEmailDialog.

On Mobile Web, the to addresses, cc addresses, bcc addresses, subject, and body are appended together and formatted into a mailto: URL. This URL typically has a limit of approximately 2000 characters.

Properties
CANCELLED [Number] Constant for the CANCELLED status result. On Android, this property exists but is not used.
FAILED [Number] Constant for the FAILED status result.
SAVED [Number] Constant for the SAVED status result. On Android, this property exists but is not used.
SENT [Number] Constant for the SENT status result.
barColor [String] Bar color of the email dialog window, as a color name or hex triplet.
bccRecipients [Array<String>] Recipients of the email included via the BCC (Blind Carbon Copy) field.
ccRecipients [Array<String>] Recipients of the email included via the CC (Carbon Copy) field.
html [Boolean] Determines whether the email message, specifically the contents of messageBody, should be sent as HTML content type rather than plain text.
messageBody [String] Email message body.
subject [String] Subject line for the email.
toRecipients [Array<String>] Recipients of the email included via the main TO field.
Events
complete Fired when this email dialog has completed sending an email.
Code Examples
Simple Email Dialog with Attachment

Create and send an email with a file attachment.

var emailDialog = Ti.UI.createEmailDialog()
emailDialog.subject = "Hello from Titanium";
emailDialog.toRecipients = ['foo@yahoo.com'];
emailDialog.messageBody = '<b>Appcelerator Titanium Rocks!</b>';
var f = Ti.Filesystem.getFile('cricket.wav');
emailDialog.addAttachment(f);
emailDialog.open();

Event properties

error [String]

Error message generated during an email send failure, or null if successful.
result [Number]

Status of the email send process. One of SENT, FAILED, SAVED (iOS only) or CANCELLED (iOS only).

Note that on Android this property will return SENT when the user discards or saves a message.
source [Object]

Source object that fired the event.
success [Boolean]

Indicates whether sending the email was successful.
type [String]

Name of the event fired.
