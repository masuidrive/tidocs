Object CreateBufferArgs

  • Android(1.7+), iPhone(1.7+), iPad(1.7+), Mobile Web(1.8+)

Arguments to be passed to createBuffer

Properties
byteOrder [Number] The byte order of this buffer.
length [Number] The length of the buffer.
type [String] The type of data encoding to use with value.
value [StringNumber] An initial value which will be encoded and placed in the buffer. If value is a Number, type must also be set. (this is simply a convenient way of calling Titanium.Codec.encodeString or Titanium.Codec.encodeNumber and placing the encoded value in the returned buffer.)
